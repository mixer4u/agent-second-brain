## Termux: установка git и клонирование репо
Пользователь устанавливает git в Termux для клонирования agent-second-brain.
- Установка прервана: нажал Y после закрытия промпта (Abort)
- Нужно повторить: `pkg install git` и сразу нажать Y
- Затем: `git clone` репозитория

### Решение
1. `pkg install git` — на вопрос [Y/n] сразу нажать Enter или Y
2. `git clone https://github.com/smixs/agent-second-brain.git`
3. Для отдельной папки Obsidian — открыть в Obsidian только папку `vault/`:
   - Путь в Termux: `~/agent-second-brain/vault`
   - Или создать symlink: `ln -s ~/agent-second-brain/vault ~/ObsidianVault`

### Синхронизация через Termux
```
cd ~/agent-second-brain && git pull
```
Для отправки изменений из Obsidian обратно:
```
cd ~/agent-second-brain && git add vault/ && git commit -m "sync from phone" && git push
```

## Android/data — Access Denied (Total Commander)
На скриншоте: Total Commander не может открыть `/storage/emulated/0/Android/data` — "Access denied! Unfortunately access to this folder has been blocked to all user-installed apps."

### Причина
Начиная с Android 11 (API 30), Google ввёл Scoped Storage — папки `Android/data` и `Android/obb` заблокированы для сторонних приложений, включая файловые менеджеры.

### Решения
1. **Termux не затронут** — Termux хранит данные в `/data/data/com.termux/`, а не в `Android/data`. Клонированный репо будет в `~/agent-second-brain/` — доступ есть.
2. **Obsidian на Android** — хранит vault в своей внутренней папке или в выбранной пользователем директории (не в Android/data). Открыть vault: указать путь `~/agent-second-brain/vault` или `/storage/emulated/0/` (домашняя).
3. **Для доступа к Android/data** (если реально нужно):
   - Через встроенное приложение "Файлы" (Files by Google)
   - Через ADB: `adb shell ls /sdcard/Android/data/`
   - Total Commander с SAF-плагином (Storage Access Framework)
4. **Для git-репо на телефоне** — лучше клонировать в домашнюю папку Termux и создать symlink в доступное место.

## Листы общих данных КМД на фасад НВФ
Согласно ГОСТ 21.502-2016 и практике проектирования навесных вентилируемых фасадов, на листах общих данных КМД указывают:

### Обязательный состав:
1. **Ведомость рабочих чертежей** — перечень всех листов комплекта КМД
2. **Ведомость ссылочных и прилагаемых документов** — ГОСТы, СП, ТУ, альбомы техрешений системодержателя
3. **Общие указания** — условия монтажа, требования к антикоррозийной защите, допуски, марки сталей и алюминиевых сплавов, требования к сварке/крепежу
4. **Ведомость спецификаций** (при нескольких спецификациях в комплекте)
5. **Условные обозначения** — обозначения элементов подконструкции, типов кронштейнов, направляющих, кляммеров

### Специфика НВФ:
- Тип фасадной системы (системодержатель, номер ТС/ТО)
- Тип и толщина утеплителя, ветрозащитной мембраны
- Тип облицовки (керамогранит, композит, фиброцемент и др.)
- Привязка к разделу КМ (марка проекта)
- Нагрузки на фасадную подконструкцию (ветровая, собственный вес)
- Требования к огнестойкости и пожарной безопасности
- Указания по монтажу (последовательность, температурные ограничения)

Источники: ГОСТ 21.502-2016, ГОСТ Р 21.101-2020, СП 522.1325800.2023
